What is Pylint?

Pylint is a static code checker for python that checks for errors, shows when you stray from a coding standard and looks for code “smells”. A “code smell” is something that “seems” wrong, such as a method that's too long or doesn't have any methods. This is basically a warning that needs to be looked into, however, most times what smells wrong is wrong.

Pylint Tutorial - Write Cleaner Python

What Does Pylint Test?

The default standard that Pylint checks the code to is PEP8. Some common errors in Pylint are “line-too-long” which means that the line length threshold has been exceeded, “trailing-whitespace” which means there are spaces at the end of a line, “len-as-condition” which means that you shouldn't use “if len(x) > 0:”, you should use “if x:”. Pylint will let you know if you don't have comments in your files, classes and methods.

Glossary of OOP terms

Factory: A Factory Design Pattern is a way to design your code so that objects are created without exposing the logic to the client. A factory method can create multiple types of objects. It takes an input, and based on the input it creates a type of object.

Instantiation: Instantiation is the creation of an object by calling its constructor.

Object: An object is an instance of a class. The object has its own copy of the class properties and methods.

Class: A class is a container for properties and methods. Class methods and properties may be accessed by an object of that class.

Namespace: Namespace is the scope of a property or method.

Constructor: A constructor is the method that instantiates an object.

Fixture: A fixture is a method that can be passed to a test method to setup the conditions for the test.

Type hint: A type hint specifies the type of a variable or the return type of a method. Below, str specifies the type of name as a string, and “→ str” specifies the return type as a string.

Type cast: A type cast converts a variable of one type to another type. “str(456) will convert the integer 456 to a string.

Unit test: unittest is a module in python's library which contains tools for testing your code. Unit testing involves creating a test function which calls the application method, passes parameters if appropriate, and tests that the return value is correct.

Static: A static method in python is a method that can be called without creating an object. Static methods are good for utility functions. They are totally self-contained and only work with data passed in as arguments.

Class method: A class method in python can also be called without creating an object. Class methods are methods that are related to a class and have access to all class-specific data.

Facade: Facade Design Pattern provides a high level interface that multiple subsystems can use. The Facade hides complexity of the system and provides a simple interface to to the client. Calculator class is a facade for calculations. Calculator wraps calculations class.

Design Patterns:

Factory Pattern: Allows you to create objects without first specifying what type of object you are creating. The Factory Method will create the appropriate object.

Builder Pattern: Requires you to builds a complex object by using a step by step approach. A Builder interface defines the steps to build the final object.

Abstract Factory Pattern: The abstract factory pattern is used to provide a client with a set of related or dependant objects. The "family" of objects created by the factory are determined at run-time.

Prototype Pattern: Create a new object instance from another similar instance and then modify according to our requirements.

The Singleton Pattern: Ensures that only one instance of a class is instantiated. All references to objects of a singlton class refer back to the same instance.

Method: A method is a container for a group of commands that perform one function. A method can be called from another method that needs that particular function performed.

Property: The Property Decorator allows you to access a class method as you would a class attribute. No () are needed. https://www.youtube.com/watch?v=jCzT9XFZ5bw
